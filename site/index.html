# AGENT-FIRST DEVELOPER OPERATIONS

Conduit turns noisy developer tools into compact, structured facts for humans, agents, scripts, and IDEs.

Modern engineering work crosses test runners, build tools, logs, OpenAPI docs, Git worktrees, service catalogs, dashboards, and databases. Conduit sits above those systems as a small, product-neutral command layer.

No hidden orchestration. No company-specific workflow baked into core. Just stable primitives that other tools can compose. 

```
$ conduit git status --json
$ conduit logs errors api --since 30m
$ conduit test rerun gradle --failed
$ conduit plugin check --provider logs
```

## PRIMITIVES

Conduit exposes factual building blocks. The core owns command semantics, validation, output rendering, and durable state. Domain behavior stays testable in library modules. 

+-----------+----------------------------------------------------------+
| COMMAND   | WHAT IT NORMALIZES                                       |
|-----------|----------------------------------------------------------|
| git       | branch, cleanliness, ahead/behind, changed files         |
| worktree  | local worktree discovery                                 |
| test      | Gradle output, JUnit XML, failures, rerunnable state     |
| logs      | filters, time windows, errors, watch/wait loops          |
| openapi   | service operations, methods, paths, parameters           |
| db        | constrained service-owned operational resources          |
| plugin    | provider metadata, contracts, capability boundaries      |

## OUTPUT MODEL

Human-readable output is a rendering of structured facts, not the source of truth.

  • compact text by default;
  • JSON for bounded command results;
  • JSONL for streams and watches;
  • raw logs preserved under local Conduit state when needed;
  • deterministic fields for agents and scripts.
runner: gradle
mode: unit
termination: exit
test_outcome: executed
status: passed
tests_ran: 1
failures: 0

## PLUGINS

Company systems belong behind explicit adapters. Conduit uses WebAssembly components for provider plugins and grants capabilities deliberately: HTTP hosts, file reads, user-scoped secrets, and other narrow permissions.

## INSTALL

Homebrew is the intended install path for released builds. Once the tap is published: 

brew tap Riki1312/conduit
brew install conduit
conduit about

The repository also pins Rust 1.94.0 for source installs.

cargo install --path crates/conduit-cli --locked
conduit about
conduit test run gradle --help
